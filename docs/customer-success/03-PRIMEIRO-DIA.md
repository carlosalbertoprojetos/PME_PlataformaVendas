# 03 - Primeiro dia

## Objetivo

Guiar a empresa no primeiro dia de uso para chegar rapidamente ao primeiro valor: produto, cliente, oportunidade, pedido ou proxima acao.

## Publico

Dono, gestor, administrador e vendedor que estao comecando a usar a plataforma.

## Quando utilizar

Use no dia da implantacao, durante o treinamento inicial ou ao preparar uma empresa piloto.

## Passo a passo

1. Confira a empresa no topo do sistema.
2. Confira se o usuario correto esta logado.
3. Acesse o dashboard.
4. Veja o checklist de onboarding.
5. Cadastre ou revise o primeiro produto.
6. Cadastre o primeiro cliente ou lead.
7. Crie a primeira oportunidade.
8. Registre uma proxima acao.
9. Compartilhe o catalogo por WhatsApp.
10. Abra o Workspace do cliente.
11. Acompanhe recomendacoes e historico.
12. Se houver pedido existente, abra e confira.
13. Volte ao dashboard para ver o progresso.

## Exemplos

- Produto: "Kit inicial de limpeza".
- Cliente: "Mercado Sao Jose".
- Oportunidade: "Reposicao mensal".
- Proxima acao: "Enviar catalogo hoje as 15h".

## Boas praticas

- Use dados reais de uma venda simples.
- Nao tente cadastrar todos os produtos no primeiro dia.
- Escolha um cliente que realmente pode comprar.
- Sempre termine uma interacao com uma proxima acao.
- Use o dashboard para conferir se a empresa esta evoluindo.

## Erros comuns

- Comecar por configuracoes em vez de cliente real.
- Criar oportunidade sem cliente.
- Compartilhar catalogo sem produto.
- Nao preencher telefone do cliente.
- Achar que pedido completo ja esta automatizado quando a fase ainda for RC1.

## Checklist

- [ ] Empresa conferida.
- [ ] Usuario conferido.
- [ ] Primeiro produto revisado.
- [ ] Primeiro cliente ou lead cadastrado.
- [ ] Primeira oportunidade criada.
- [ ] Primeira proxima acao criada.
- [ ] Catalogo compartilhado.
- [ ] Workspace aberto.
- [ ] Dashboard revisado.
