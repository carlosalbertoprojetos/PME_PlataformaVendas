# 40 - README Customer Success

## Objetivo

Explicar como usar toda a documentacao de Customer Success.

## Publico

Equipe interna, suporte, CS, comercial, gestores e usuarios finais.

## Quando utilizar

Use como ponto de entrada para encontrar o documento certo.

## Passo a passo

1. Se for novo usuario, comece em Bem-vindo.
2. Se for implantacao, use Primeiro dia e Playbook implantacao.
3. Se for vendedor, use Guia do vendedor e Checklist diario.
4. Se for gestor, use Guia do gestor e Dashboard.
5. Se for suporte, use Resolucao de problemas e Playbook suporte.
6. Se for CS, use Jornada, Health Score, Retencao e Expansao.

## Exemplos

- Duvida simples: FAQ.
- Problema operacional: Resolucao de problemas.
- Treinamento: Academia.
- Evolucao do cliente: Jornada do cliente.

## Boas praticas

- Usar links relacionados.
- Atualizar docs quando produto mudar.
- Manter linguagem simples.
- Nao prometer recurso futuro como se ja existisse.

## Erros comuns

- Usar documento tecnico com usuario leigo.
- Duplicar conteudo em vez de referenciar.
- Nao atualizar limitacoes do RC1.
- Esquecer de treinar suporte.
