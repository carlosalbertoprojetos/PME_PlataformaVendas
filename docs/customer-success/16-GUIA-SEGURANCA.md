# 16 - Guia Seguranca

## Objetivo

Ensinar boas praticas simples para proteger acessos, dados de clientes e operacao comercial.

## Publico

Donos, administradores, gestores, vendedores e suporte.

## Quando utilizar

Use na implantacao, ao criar usuarios, ao revisar acessos ou quando houver suspeita de uso indevido.

## Passo a passo

1. Crie usuario individual para cada pessoa.
2. Use papel correto: administrador ou vendedor.
3. Oriente sobre senha forte.
4. Remova acesso de quem saiu da empresa.
5. Confira se o usuario esta na empresa correta.
6. Evite compartilhar prints com dados de clientes.
7. Avise suporte ao notar acesso estranho.

## Exemplos

- Vendedor novo: recebe usuario proprio.
- Vendedor desligado: acesso deve ser removido.
- Gestor que administra plano: pode ser admin.

## Boas praticas

- Nunca compartilhe senha.
- Revise usuarios todo mes.
- Use admin apenas para quem precisa.
- Nao use dados reais em ambiente demo.
- Tenha cuidado ao enviar relatorios por WhatsApp.

## Erros comuns

- Todos usando o mesmo login.
- Senha anotada em local publico.
- Ex-funcionario ainda ativo.
- Admin concedido sem necessidade.
- Tratar ambiente local como producao.

## Checklist

- [ ] Cada pessoa tem usuario proprio.
- [ ] Admins foram revisados.
- [ ] Vendedores ativos conferidos.
- [ ] Senhas nao sao compartilhadas.
- [ ] Acessos antigos foram removidos.
