# 31 - Playbook suporte

## Objetivo

Definir um fluxo simples para atender usuarios, resolver problemas e escalar casos tecnicos.

## Publico

Suporte, Customer Success, administradores e equipe tecnica.

## Quando utilizar

Use sempre que um usuario relatar duvida, erro, bloqueio, limite ou dificuldade operacional.

## Passo a passo

1. Receba o chamado.
2. Confirme empresa, usuario e tela.
3. Pergunte o que o usuario tentou fazer.
4. Verifique se e duvida, permissao, dado faltando ou erro.
5. Consulte a resolucao de problemas.
6. Oriente o usuario.
7. Se for tecnico, escale com evidencias.
8. Registre a resposta.

## Exemplos

- Duvida: "Nao vejo WhatsApp".
- Permissao: "Nao vejo Plano".
- Dado faltando: "Catalogo vazio".
- Tecnico: erro inesperado em tela.

## Boas praticas

- Responder com linguagem simples.
- Pedir print quando necessario.
- Nao prometer prazo sem alinhamento.
- Confirmar se o usuario conseguiu seguir.
- Registrar aprendizado em FAQ.

## Erros comuns

- Escalar sem informacoes.
- Culpar usuario.
- Usar termos tecnicos.
- Resolver sem ensinar.
- Ignorar recorrencia.

## Checklist

- [ ] Empresa identificada.
- [ ] Usuario identificado.
- [ ] Tela identificada.
- [ ] Problema classificado.
- [ ] Solucao orientada.
- [ ] Escalado se necessario.
