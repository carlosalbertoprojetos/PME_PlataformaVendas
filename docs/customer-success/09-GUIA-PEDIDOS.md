# 09 - Guia Pedidos

## Objetivo

Explicar o fluxo de pedidos e deixar claro o que esta disponivel no RC1 e o que faz parte da evolucao para producao comercial.

## Publico

Gestores, vendedores e suporte.

## Quando utilizar

Use quando precisar consultar pedido existente, compartilhar pedido por WhatsApp ou entender como pedidos entram nos indicadores.

## Passo a passo

1. Abra o cliente ou Workspace.
2. Procure pedidos anteriores.
3. Abra o detalhe do pedido.
4. Confira cliente, vendedor, status e total.
5. Confira itens do pedido.
6. Compartilhe pedido por WhatsApp se houver necessidade.
7. Volte ao Workspace para registrar proxima acao.

## Exemplos

- Pedido pendente: confirmar com o cliente se deseja fechar.
- Pedido confirmado: entra em receita e ticket medio.
- Pedido cancelado: nao deve ser tratado como venda concluida.

## Boas praticas

- Confira o status antes de usar o pedido em uma conversa.
- Use pedido confirmado para analise de receita.
- Registre follow-up depois de compartilhar pedido.
- Nao misture pedido com oportunidade: oportunidade e negociacao; pedido e intencao registrada.

## Erros comuns

- Contar pedido pendente como receita.
- Enviar pedido para cliente errado.
- Nao conferir itens.
- Esperar criacao completa de pedido pela UI quando a fase atual ainda estiver limitada.

## Checklist

- [ ] Cliente do pedido conferido.
- [ ] Status conferido.
- [ ] Total conferido.
- [ ] Itens conferidos.
- [ ] Compartilhamento feito quando necessario.
- [ ] Proxima acao registrada.
