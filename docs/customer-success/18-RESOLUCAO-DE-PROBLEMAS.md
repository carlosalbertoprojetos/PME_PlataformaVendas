# 18 - Resolucao de problemas

## Objetivo

Ajudar usuarios e suporte a resolver problemas comuns sem linguagem tecnica.

## Publico

Usuarios, administradores, suporte e implantacao.

## Quando utilizar

Use quando algo nao funcionar como esperado ou quando o usuario nao souber o proximo passo.

## Passo a passo

1. Identifique a tela.
2. Descreva o que tentou fazer.
3. Leia a tabela abaixo.
4. Aplique a solucao simples.
5. Se nao resolver, acione suporte com print e horario.

## Exemplos

- "Nao vejo o botao Plano": talvez voce nao seja admin.
- "WhatsApp nao aparece": talvez cliente nao tenha telefone valido.
- "Nao consigo acessar cliente": talvez seja de outra empresa.

## Tabela de problemas

| Problema | Possivel causa | Como resolver |
| --- | --- | --- |
| Nao consigo entrar | Usuario ou senha incorretos | Confira dados e acione administrador |
| Vejo empresa errada | Usuario vinculado incorretamente | Acione administrador |
| Nao vejo Plano | Usuario nao e admin | Peça acesso ou fale com admin |
| Nao vejo WhatsApp | Telefone ausente ou invalido | Atualize telefone do cliente |
| Catalogo vazio | Sem produtos ativos | Cadastre produto ativo |
| Recomendacao nao aparece | Nao ha dados suficientes | Cadastre clientes, pedidos e oportunidades |
| Oportunidade nao aparece | Pode ser de outra empresa | Confirme empresa e permissao |
| Cliente nao abre | Cliente inativo ou sem permissao | Verifique cadastro |
| Pedido nao abre | Pedido de outra empresa ou inexistente | Confira link e empresa |
| Plano bloqueia criacao | Limite atingido | Avalie upgrade |
| Automacao falhou | Regra ou dado incompleto | Verifique log e acione suporte |
| Dashboard sem numeros | Empresa nova sem dados | Siga onboarding |
| Proxima acao nao aparece | Data/status diferente | Veja agenda completa |
| Link de WhatsApp errado | Telefone cadastrado errado | Corrija telefone |
| Produto duplicado | Cadastro repetido | Revise produtos antes de criar |
| Historico confuso | Resumo pouco claro | Registre contato com texto objetivo |
| Vendedor nao aparece | Usuario sem membership ativo | Admin deve revisar usuario |
| Limite proximo | Uso atingiu faixa de aviso | Planeje upgrade ou limpeza |
| Pedido nao pode ser criado | Fluxo ainda limitado no RC1 | Use acompanhamento manual e registre proxima acao |
| Erro 403 | Sem permissao | Peça acesso correto |
| Erro 404 | Item nao existe ou nao e da sua empresa | Confira link e empresa |

## Boas praticas

- Sempre informe empresa e usuario.
- Tire print antes de sair da tela.
- Nao tente burlar permissao.
- Registre o problema com palavras simples.
- Diferencie erro de sistema de dado faltando.

## Erros comuns

- Relatar "nao funciona" sem contexto.
- Usar usuario de outra pessoa.
- Criar dado duplicado em vez de procurar.
- Ignorar mensagem de limite.

## Checklist

- [ ] Identifiquei a tela.
- [ ] Identifiquei usuario e empresa.
- [ ] Li a tabela.
- [ ] Tentei a solucao simples.
- [ ] Enviei print ao suporte se necessario.
